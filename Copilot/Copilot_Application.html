About GitHub Student Developer Pack

The GitHub Student Developer Pack includes GitHub Pro and GitHub Copilot benefits. It also enables you to apply for various software subscriptions like Termius, limited-time free credits from cloud service providers like Azure, and one-year free domain registrations for specific TLDs (Top-Level Domains, e.g., .com, .org, .me) from participating registrars.

Detailed Application Steps

1Ô∏è‚É£

Step 1: Create or Log in to GitHub Account

  1. Visit GitHub official website
  2. If you don't have an account, click "Sign up" to create one
  3. If you already have an account, log in directly

Recommendation: Register with your school email to increase verification success rate.

2Ô∏è‚É£

Step 2: Enable Two-Factor Authentication (2FA)

  1. Before applying for student verification, complete  GitHub's 2FA authentication
  2. Click "Enable two-factor authentication" and follow the instructions. Remember to save your recovery codes file (github-recovery-codes.txt)
3Ô∏è‚É£

Step 3: Apply for GitHub Student Developer Pack

  1. After logging in, visit GitHub Education
  2. Select the "Student" option
  3. Choose your email address (preferably an educational email with .edu.cn domain)
  4. Enter your school's full name (no abbreviations), e.g., Nanjing University
  5. Explain how you plan to use GitHub (this isn't critical). You can use this sample reason: "I want to learn coding and try to make contribution to the community."
  6. Click the green "Continue" button to proceed to the next page
  7. Upload your prepared materials
  8. In "What is your school's website", enter your school's official URL (http:// or https:// prefix doesn't matter)
  9. In "How would you describe your school?", select your institution type: choose "Higher-education" for universities
  10. In "Street address", enter your school's street/district location
  11. In "City", enter your city name, e.g., Nanjing/Nanjing City
  12. In "Country", select your country, e.g., China
  13. In "Non-US State or CA Province", enter your province, e.g., Jiangsu
  14. Click the green "Process my application" button to submit

Recommendation: If possible, use a mobile device for this step. Currently GitHub appears to require camera access for document uploads rather than file selection. Ensure your photos are in focus and clearly legible.

Important: During the entire application process, you must maintain location permissions for GitHub (required) and avoid using proxies (except for Bing Maps when GitHub requires map selection - in this case you may proxy these domains: www.bing.com, bing.com, cn.bing.com).

Step 4: Wait for Review

  • Review typically takes 3-5 business days
  • You'll receive email notification from GitHub
  • You can check application status during this period

Step 5: Activate GitHub Copilot

  1. After approval, visit  Copilot settings page
  2. Select "GitHub Copilot"
  3. Follow the setup instructions
  4. Install Copilot plugin for your IDE (VS Code, JetBrains, etc.)

Congratulations! You can now start using GitHub Copilot for programming learning.

What if my application is rejected?

  • Check if submitted materials are clear and complete
  • Confirm student status is still valid
  • Try contacting GitHub support team
  • You may resubmit with more complete proof materials

How to renew student eligibility?

  • You'll receive a reminder email 30 days before expiration
  • Resubmit current student proof
  • Renewal review is typically faster than initial application
